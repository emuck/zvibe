# Servant ZVibe Firmware

Firmware and support libraries for the Servant ZVibe RISC-V SoC.
Targets the MAX10 08 Evaluation Board (UFM XIP) and Arty S7-50 (QSPI XIP).

## Source Files

### Main Firmware
- **zvibe_riscv_multi.c** — Main firmware: multi-game ZVibe interpreter with XIP, auto-restart
- **zvibe_stubs.c** — Platform stubs: UART I/O, memory access

### Startup and Runtime
- **xip_start.S** — XIP startup: copies .ramfunc and .data, jumps to main
- **crt0_minimal.S** — Minimal C runtime for RAM-based programs

### Game Registry
- **game_registry_riscv.c/h** — Reads ZVGM TOC from flash at boot to discover games
- **menu_system.c/h** — Text menu for multi-game selection

### Save System
- **flash_save.c/h** — Platform-agnostic save system with wear-leveling slot rotation
- **flash_hal.h** — Common flash HAL interface (implemented per-board in board sw/)
- **flash_metadata.c/h** — Reads ZVIF metadata header from flash
- **save_delta.c/h** — Delta compression for compact save states

Board-specific HAL implementations live in `boards/*/sw/`:
- `boards/arty_s7_50/sw/flash_driver.c` — Arty QSPI flash driver (HAL implementation)
- `boards/max10_08_eval/sw/flash_driver_max10_ufm.c` + `flash_hal_max10.c` — MAX10 UFM driver

### Linker Scripts
- **xip_link.ld** — XIP programs (code in flash, data in RAM)
- **link.ld** — RAM-based programs (32KB @ 0x00000000)
- **flash_boot_link.ld** — Boot stub (<1KB, for BRAM init)

### Test Programs
- **uart_echo.c** — UART echo test (RAM-based)
- **uart_echo_xip.c** — UART echo from XIP flash

### Build Tools
- **zvibe_flash_lib.py** — Shared ZVIF/ZVGM serialization library
- **create_flash_image.py** — Flash image assembly utility
- **bin2c.py** — Binary to C byte array converter

### Generated (do not edit)
- **flash_layout.h** — Flash offsets, generated by `build_flash.py`
- **max10_layout.json** — Layout calculations, generated by `build_flash.py`

## Build Commands

```bash
# MAX10 firmware + flash image (firmware + ZVIF header + ZVGM TOC + game)
make max10-flash-test

# MAX10 firmware binary only
make zvibe_riscv_multi_max10.bin

# Preview flash layout without building
make check-layout

# Clean
make clean

# Show all targets
make help
```

## Flash Layout

Offsets are computed dynamically by `boards/max10_08_eval/build_flash.py`
based on actual binary sizes, then written to `flash_layout.h`.

```
Physical    Virtual       Content
0x000000    0x80000000    ZVIF metadata header (256B)
0x000100    0x80000100    Firmware (reset PC, ~37KB)
varies      varies        ZVGM TOC v2 (16B header + 80B per entry)
varies      varies        Game data (up to 126KB)
varies      varies        Save slots (2KB each; offset in TOC entry)
```

## Memory Map

| Address | Size | Description |
|---------|------|-------------|
| 0x00000000 | 32KB | RAM (data, stack, .ramfunc sections) |
| 0x40000000 | 16B | UART peripheral |
| 0x40000010 | 16B | Timer peripheral |
| 0x40000020 | 16B | GPIO LEDs |
| 0x40000030 | 16B | Flash status (read-only) |
| 0x40000040 | 32B | UFM write controller |
| 0x80000000 | 172KB | UFM XIP region |

## Platform Differences

| Platform | Flash | Base Address | Boot |
|----------|-------|--------------|------|
| MAX10 | UFM (172KB on-chip) | 0x80000100 | Pure XIP |
| Arty S7-50 | QSPI (16MB external) | 0x80400000 | Boot stub + XIP |

The `__flash_origin` linker symbol sets the code base address.
All other firmware source is shared between platforms.

## Critical: objcopy Format

```bash
# For simulation .dat files (word-addressed, with address adjustment)
riscv64-unknown-elf-objcopy -O verilog --verilog-data-width=4 \
  --change-addresses=-0x80000000 firmware.elf altera_onchip_flash.dat

# For hardware binary (byte-addressed)
riscv64-unknown-elf-objcopy -O binary firmware.elf firmware.bin
```

Wrong format causes the CPU to execute garbage data.
