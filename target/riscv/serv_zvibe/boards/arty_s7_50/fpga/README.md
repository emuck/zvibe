# Servant ZVibe FPGA Build System (Arty S7-50)

This directory contains the FPGA build system for Servant ZVibe on the Digilent Arty S7-50.

## Files

- **Makefile** — Build automation (`make help` for all targets)
- **build.tcl** — Vivado synthesis + implementation + bitstream script
- **arty_s7_50.xdc** — Pin constraints and timing constraints (166MHz)

## Quick Start

```bash
# Build FPGA bitstream (~5-10 min)
make build

# Build firmware + flash image, then program everything
make program-complete          # single-game (Plundered Hearts default)
make program-complete ARTY_FLASH=../sw/arty_flash_all.bin   # all 24 games

# Update flash only (bitstream unchanged)
make program-flash             # fast firmware/game update (~30s)

# View timing & utilization reports
make reports

# Clean build outputs
make clean
```

## Hardware Configuration

**FPGA:** Xilinx Spartan-7 XC7S50-CSGA324-1
- Clock: 166 MHz (100 MHz MMCM input × 5/3)
- Reset: BTN0 active-high (G15)
- Timing: WNS = +0.288ns @ 166MHz

**RAM:** 32KB BRAM (0x00000000–0x00007FFF)

**UART:** 115200 baud, 8N1
- TX: R12 (FPGA → PC via FT2232H channel B)
- RX: V12 (PC → FPGA)
- Port: `/dev/ttyUSB1` or `/dev/ttyUSB2` depending on other connected USB devices

**Flash:** S25FL128S 16MB QSPI
- CS#: L13 / DQ[3:0]: K17, K18, L14, M14 / CLK: via STARTUPE2

**GPIO LEDs:**
- LD2 (H15): Heartbeat
- LD3 (F13): gpio_led[0]
- LD4 (E13): gpio_led[1]
- LD5 (G16): gpio_led[2]

## Flash Layout

Programming writes two regions:

| Step | Tool | Offset | Content |
|------|------|--------|---------|
| 1 | openFPGALoader | 0x000000 | FPGA bitstream |
| 2 | openFPGALoader | 0x100000 | ZVibe user region (ZVIF + firmware + games) |

The user region is built by `../sw/Makefile` (see `make flash-test` or `make all-games`).

## Build Artifacts

Generated by `make build` / `build.tcl`:

- `build/timing_summary.txt` — Timing analysis
- `build/utilization_*.txt` — Resource utilization

## Troubleshooting

**No UART output:**
1. Check UART port: `ls /dev/ttyUSB*`
2. Connect: `picocom -b 115200 /dev/ttyUSBN`
3. FT2232H UART is the second channel — highest-numbered port
4. Reprogram: `make program-complete`

**Timing violations:**
- Check `build/timing_summary.txt` (WNS ≥ 0 required)
- MMCM reset is a false path (set in XDC) — ignore related warnings
